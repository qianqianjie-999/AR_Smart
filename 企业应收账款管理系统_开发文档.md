## 1. 项目概述

### 1.1 项目背景
企业目前使用Excel表格管理应收账款，存在以下痛点：

| 痛点 | 具体表现 | 系统解决方案 |
|---|---|---|
| 数据分散 | 多人协作困难，版本混乱 | 集中式B/S架构，统一数据源 |
| 付款进度跟踪难 | 多阶段付款节点靠人工记忆 | 付款节点自动提醒，逾期标红 |
| 诉讼时效管理缺失 | 超时效导致无法追款 | 自动计算时效，三级预警机制 |
| 客户信息未集中 | 联系人、地址散落在各表格 | 客户档案集中管理，多联系人支持 |
| 缺乏数据分析 | 无法快速统计欠款、账龄 | 数据看板+多维度报表 |

### 1.2 建设目标
构建一套B/S架构的应收账款管理系统，实现四大核心管理：

1. **资料集中管理**：合同、验收、结算、发票、催款函等附件统一管理
2. **付款进度管理**：多阶段付款节点跟踪，到期预警，逾期提醒
3. **中断时效管理**：自动计算3年诉讼时效，回款/催款自动中断时效，到期前90/30/7天三级预警
4. **客户信息管理**：客户档案、多联系人（第一/第二/第三/其他）、区域划分

### 1.3 技术选型

| 层级 | 技术方案 | 版本 | 选型理由 |
|---|---|---|---|
| **前端框架** | Vue3 | 3.4+ | Composition API，性能优异，生态成熟 |
| **UI组件库** | Element Plus | 2.5+ | 管理后台标配，组件丰富，中文文档完善 |
| **状态管理** | Pinia | 2.1+ | Vue官方推荐，TypeScript友好 |

| **HTTP客户端** | Axios | 1.6+ | 拦截器支持，请求统一处理 |
| **构建工具** | Vite | 5.0+ | 极速冷启动，HMR热更新 |

| **密码加密** | bcrypt | 4.1+ | 安全哈希 |

## 2. 需求分析

### 2.1 用户角色与权限

| 角色 | 职责描述 | 可操作模块 | 数据范围 |
|---|---|---|---|
| **系统管理员** | 系统配置、用户管理、数据备份 | 全部 | 全部 |
| **财务人员** | 回款登记、开票管理、账龄分析 | 回款、开票、报表 | 全部 |
| **业务人员** | 合同录入、客户维护、催款跟进 | 客户、合同、催款 | 全部 |
| **法务人员** | 时效监控、催款函管理、诉讼跟踪 | 时效、催款、报表 | 全部 |
| **部门经理** | 数据查看、审批、报表分析 | 查看全部，审批回款/催款 | 全部 |
| **普通查看** | 仅查看数据，无操作权限 | 仅查看 | 全部 |

### 4.2 数据表详细设计

#### 4.2.1 客户表 (customer)
```sql
CREATE TABLE customer (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    region          VARCHAR(50) NOT NULL COMMENT '区域（如：济西、济南、青岛等）',
    name            VARCHAR(200) NOT NULL COMMENT '客户名称',
    registered_addr VARCHAR(500) COMMENT '客户注册地址',
    contact_addr    VARCHAR(500) COMMENT '客户联系地址',
    billing_info    TEXT COMMENT '开票信息（税号、开户行、账号、地址、电话等JSON格式）',
    business_contact VARCHAR(100) COMMENT '业务联系人姓名',
    credit_level    VARCHAR(20) DEFAULT 'B' COMMENT '信用等级 A/B/C/D',
    remark          TEXT COMMENT '备注',
    is_deleted      TINYINT DEFAULT 0 COMMENT '是否删除 0否 1是',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    created_by      BIGINT COMMENT '创建人ID',
    updated_by      BIGINT COMMENT '更新人ID',
    INDEX idx_region (region),
    INDEX idx_name (name),
    INDEX idx_credit (credit_level)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='客户表';
```

#### 4.2.2 客户联系人表 (customer_contact)

```sql
CREATE TABLE customer_contact (
```

```sql

    total_paid          DECIMAL(18,2) DEFAULT 0 COMMENT '累计回款金额（元）',

    is_deleted          TINYINT DEFAULT 0 COMMENT '是否删除 0否 1是',

    updated_at          DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    created_by          BIGINT COMMENT '创建人ID',
    updated_by          BIGINT COMMENT '更新人ID',
```

#### 4.2.4 付款节点表 (payment_node)
```sql
CREATE TABLE payment_node (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    contract_id     BIGINT NOT NULL COMMENT '关联合同ID',
    node_no         INT NOT NULL COMMENT '节点序号（1,2,3...）',

    pay_ratio       DECIMAL(5,2) COMMENT '付款比例（%）',
    pay_amount      DECIMAL(18,2) COMMENT '应付金额（元）',
    due_date        DATE COMMENT '应付时间',
    due_condition   TEXT COMMENT '付款条件（如：验收合格后30日内）',
    actual_pay_date DATE COMMENT '实际付款日期',
    actual_pay_amount DECIMAL(18,2) DEFAULT 0 COMMENT '实际付款金额（元）',
    status          VARCHAR(20) DEFAULT '未到期' COMMENT '状态：未到期/已到期/已付款/逾期',
    remark          TEXT COMMENT '备注',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (contract_id) REFERENCES contract(id) ON DELETE CASCADE,
    INDEX idx_contract (contract_id),
    INDEX idx_status (status),
    INDEX idx_due_date (due_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='付款节点表';
```

#### 4.2.5 回款记录表 (payment_record)
```sql
CREATE TABLE payment_record (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',

    amount          DECIMAL(18,2) NOT NULL COMMENT '回款金额（元）',
    payment_date    DATE COMMENT '回款时间',
    payment_method  VARCHAR(50) COMMENT '回款方式：银行转账/银行承兑/现金/其他',
    bank_account    VARCHAR(100) COMMENT '回款银行账号（加密存储）',
```

```python
def import_from_excel(file_path: str):
    import pandas as pd

    df = pd.read_excel(file_path, header=2)

    for _, row in df.iterrows():
        # 1. 创建/获取客户
        customer = Customer.query.filter_by(name=row['客户名称']).first()
        if not customer:
            customer = Customer(
                region=row['区域'],
                name=row['客户名称'],
                registered_addr=row['客户注册地址'],
                contact_addr=row['客户联系地址'],
                billing_info=row['开票信息'],
                business_contact=row['业务联系人']
            )
            db.session.add(customer)
            db.session.flush()

        # 2. 创建合同
        contract = Contract(
            customer_id=customer.id,
            contract_no=row['合同编号'],
            project_name=row['项目名称'],
            sign_date=parse_date(row['签订日期']),
            acceptance_date=parse_date(row['验收日期']),
            contract_amount=parse_amount(row['合同金额']),
            audit_amount=parse_amount(row['审计（结算）金额']),
            payment_method=row['付款方式'],
            payment_terms=row['付款节点']
        )
        db.session.add(contract)
        db.session.flush()

        # 3. 解析付款节点并创建
        payment_nodes = parse_payment_terms(row['付款节点'])
        for node in payment_nodes:
            PaymentNode(
                contract_id=contract.id,
                node_no=node['node_no'],
                node_name=node['node_name'],
                pay_ratio=node['pay_ratio'],
                pay_amount=node['pay_amount'],
                due_date=node['due_date'],
                due_condition=node['due_condition']
            )

        # 4. 解析回款记录
        payments = parse_payments(row['收回货款'])
        for payment in payments:
            PaymentRecord(
                contract_id=contract.id,
                amount=payment['amount'],
                payment_date=payment['date'],
                payment_method=payment['method']
            )

        # 5. 计算尚欠金额
        contract.total_paid = sum(p['amount'] for p in payments)
        contract.outstanding_amount = contract.audit_amount - contract.total_paid

        # 6. 初始化时效记录
        limitation_result = calc_limitation_end(contract.id)
        LimitationRecord(
            contract_id=contract.id,
            base_date=limitation_result['base_date'],
            limitation_end_date=limitation_result['limitation_end_date'],
            days_remaining=limitation_result['days_remaining'],
            status=limitation_result['status']
        )

    db.session.commit()
```
